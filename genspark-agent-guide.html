Chapter 01

普通のAIと何が違う？

GenSparkは「1人のAI」に話しかけるのではなく、
AIのチーム全体がタスクに応じて自動で動く次世代プラットフォームです。

従来のチャットAI

  • 1つの質問に1つの答え
  • 複雑な作業は自分で分解が必要
  • 資料・スライド・表は別アプリ
  • タスクを直列で1つずつ処理
  • カスタマイズがほぼできない

GenSpark スーパーエージェント

  • タスクを自動で細かく分解
  • 専門エージェントを自律的に編成
  • スライド・表・文書を同時生成
  • 複数エージェントが並列で高速処理
  • 目的別カスタムエージェント作成可
🧠

リード（オーケストレーター）

メインAIがタスクを理解し、どのサブエージェントに何を任せるかを自律的に判断します。

⚡️

並列処理

複数のサブエージェントが同時に稼働。リサーチ・資料作成・データ分析を同時に進めます。

📄

Sparkpage出力

各エージェントの成果物を1つの美しいWebレポート（Sparkpage）にまとめて提示します。

Chapter 01.5

スーパーエージェントとは何か

GenSparkの核心は「1人のAI」ではなく、司令官と専門家チームの階層構造にあります。

司令塔
🧠

スーパーエージェント

プロジェクトマネージャー

全体像を把握し、タスクを分割してサブエージェントに指示を出す。あなたから受け取ったゴールを「誰に・何を・どの順番でやらせるか」に変換する役割。

実行部隊
⚡️

サブエージェント

各分野のスペシャリスト

マネージャーから割り振られた「リサーチ」「執筆」「分析」「スライド作成」などの特定タスクに集中して実行する専門家。複数が同時並列で動く。

指示の流れ

👤 あなた（CEO） ゴールを一言で伝えるだけ
↓ ゴールを伝える
🧠 スーパーエージェント タスクを分解・サブエージェントに指示
↓ 役割を分担して指示
🔍 リサーチ
✍️ 執筆
📊 分析
🖼️ スライド
✨ その他
↓ 成果を統合してレポート化
📄 Sparkpage（完成レポート） あなたの元に届く

あなたが覚えておくべきこと：
スーパーエージェントは「AIのプロジェクトマネージャー」。あなたはゴールを伝えるだけで、タスクの分解・担当割り当て・並列実行・最終統合をすべてスーパーエージェントが自律的にこなします。
Chapter 02

専門サブエージェントの種類

GenSparkには目的別の専門エージェントが揃っています。
リードAIがタスクに応じて自動で使い分けます。

🖼️

AI Slides Agent

プレゼン資料・ピッチデッキを自動生成。テーマ・構成・デザインをAIが一括で作成します。

📊

AI Sheets Agent

リサーチアナリストとして機能。競合の再生数・いいね・コメント数などをリアルタイムで自動収集してライブテーブルに整理します。

📝

AI Docs Agent

企画書・提案書・仕様書・SNSキャプションなどフォーマットされたドキュメントを自動生成。

💻

AI Developer（Code Agent）

Webサイト・アプリ・ゲームをコード生成・実行。GitHubとの連携でプルリクエストまで自動対応します。

🎨

AI Designer

FLUX・Gemini Imagen 4など複数の画像生成AIモデルを使ってビジュアルコンテンツを生成します。

🎬

AI Video Agent

テキスト→動画変換。Sora 2・Kling V2.5・Gemini Veo 3.1など14以上のモデルから最適なものを自動選択。クリップ延長・リップシンクも対応。

📞

Call for Me

AIが実際に電話をかけてレストラン予約・問い合わせを代行。成功率83%。通話内容はリアルタイムで画面表示（米国・日本の一部対応）。

🔍

並列ウェブ検索

複数ソースを同時に検索・クロスチェック。リアルタイムの市場調査・競合分析に使われます。

✨

Ephemeral Agent

タスク専用にリアルタイムで自動生成される使い捨てエージェント。リードAIが必要と判断したとき自動で作られ、完了後に消えます。

🌐

Sparkpage Agent

各エージェントの成果を統合し、広告なしの美しいWebページ（Sparkpage）として自動出力。URLで即共有できます。

Chapter 03

スーパーエージェントの仕組み

あなたが「ゴール」を伝えると、AIがプロジェクトマネージャーのように
チームを組んで仕事を動かします。

1

ゴールを入力

チャットに大まかな目標（例：「新商品のSNS戦略を作って」）を入力するだけでOK。細かい指示は不要です。

2

リードAIがタスクを分解

オーケストレーターAIがゴールを分析し、「どんなサブタスクに分けるか」「どのエージェントを使うか」を自律的に計画します。

3

サブエージェントが並列稼働

複数の専門エージェントが同時に作業開始。市場調査・スライド作成・データ整理が同時進行します。

4

成果を統合して提示

各エージェントの結果をリードAIが集約し、Sparkpageなどの形で一度に受け取れます。確認・修正して完成です。

💡

あなたの立ち位置はCEO

細かい作業指示ではなく「最初のゴール設定」と「成果物のチェック・修正」に集中するだけでOK。AIチームが自律的に動きます。

🔄

途中で軌道修正できる

実行プランが表示されたら承認・修正が可能。「このステップは不要」「このエージェントを追加して」と指示できます。

Chapter 04

カスタムエージェントの作り方

プログラミング不要。自分だけの専門AIを5分で作れます。
目的別に複数作っておくと、毎回ゼロから指示する手間がなくなります。

1

Gensparkにログイン

genspark.ai にアクセスしてアカウントにログインします。

2

「Custom Agent」を開く

ホーム画面中央または左サイドバーの「+ New（新規作成）」メニューから「Custom Agent（カスタムエージェント）」をクリックします。

3

名前と説明を入力

エージェントの名前（例：「SNSマーケター」「プロジェクト総責任者」）と、何をするAIかの説明文を入力します。

💡 名前はチームで使う場合にも分かりやすい名前にしましょう
4

指示文（Instructions）を入力

このAIにどんな役割・行動をしてほしいかを書きます。下の「プロンプト集」からコピーして貼り付けるだけでOKです。

📋 次のChapter 05のプロンプトをそのまま使えます
5

（任意）ファイルをアップロード

過去の実績・商品マニュアル・競合資料などをPDF/Word/CSV形式でアップロードすると、AIがそれを学習して精度が上がります。（→ Chapter 06 参照）

6

保存して完成

「Create / Save」ボタンを押せば完成。自分だけで使う・URLで他の人に共有する、どちらも選べます。

🔗 URLを共有すればチームメンバーも同じエージェントを使えます
ボーナス

Custom Super Agent Store

自分でゼロから作らなくても、コミュニティが公開しているカスタムエージェントをそのまま利用・コピーできるマーケットプレイスが存在します。気に入ったエージェントをフォークして自分用にカスタマイズすることも可能です。「SNSマーケター」「競合分析エージェント」「採用担当エージェント」など多様なテンプレートが揃っています。

Chapter 05

目的別プロンプト集

カスタムエージェント作成時の「Instructions（指示文）」にそのまま
コピー&ペーストして使えるテンプレートです。

新規事業・技術計画・イベント企画など幅広いジャンルでサブエージェントチームを立ち上げたいときに使います。

Instructions（指示文）
# 役割・目的 あなたはこのプロジェクトの総責任者（リードエージェント）です。 ユーザーから指示されたプロジェクトテーマに基づき、仕事量・タスクの専門性をみずから自律的に判断し、適切な専門サブエージェントをアサイン・編成して、プロジェクトを完結させてください。 # ユーザーからの入力 ユーザーがチャットに持ち込むテキストはすべて「プロジェクトテーマ」として認識してください。 # 期待する成果物 プロジェクトを効率的に達成するために、以下の要素を網羅した最終レポート（Sparkpage・データシート・スライド構成など）を作成してください。 1. 市場調査・ターゲット分析（競合の価格帯・強み・弱みの比較を含む） 2. ビジネスモデル・具体的な施策の提案（価格設定、集客・実行アイデア） 3. 今後のロードマップ（立ち上げまでの具体的なステップ） # サブエージェントへの指示（オーケストレーション方針） - 必要に応じて、Gensparkの専門機能である「並列検索（ウェブ検索エージェント）」「AI Sheet Agent（データ分析）」「AI Slides Agent（資料作成）」などを積極的に並行で裏側から起動し、タスクを並列処理してください。 - 各サブエージェントが持ち帰った個別のデータや成果物を、リードエージェントとして1つの美しい最終的な統合レポートにまとめてユーザーに提示してください。 - 情報が不足している、または判断に迷う点がある場合は、作業を進める前に自律的にユーザーに質問を促してください。 # 動作フロー ユーザーから「プロジェクトテーマ」が入力されたら、まず最初に「どのようなステップにタスクを分解し、どんなサブエージェントのチーム体制で進めるか」という実行プランを提示してください。ユーザーの承認または修正指示を得てから、各サブエージェントを起動して実際の作業に移ってください。

Instagramのビジュアル表現やハッシュタグ戦略、ストーリーズ・リールの投稿設計をサブエージェントチームで一括対応します。

Instructions（指示文）
# 役割・目的 あなたはこのプロジェクトの総責任者（リードエージェント）であり、Instagramマーケティングのエキスパートです。 ユーザーから指示された商品・サービス情報に基づき、自律的に専門サブエージェントを編成し、Instagramでファンを増やす運用・投稿プランを完結させてください。 # ユーザーからの入力 ユーザーがチャットに持ち込むテキスト（商品名・ターゲット・目的など）はすべて分析対象として認識してください。 # 期待する成果物 以下の要素を網羅した、視覚的で具体的なInstagram戦略レポート（Sparkpage・データシートなど）を作成してください。 1. 「トレンド・競合調査」：Instagramで現在バズっているリール動画の傾向、競合アカウントのビジュアル表現、ターゲットに刺さるハッシュタグの調査 2. 「アカウント世界観・ペルソナ」：発信の軸（テーマ）、プロフィール文の改善案、グリッド（投稿一覧）の統一感に関するヒント・設計 3. 「フィード・リールズ・ストーリーズ投稿案」：今すぐ投稿できるキャプション（文章）テキスト案（最低3パターン）、ハッシュタグの選定（大・中・小キーワード）、投稿画像/動画の構成・レイアウト指示（ビジュアルイメージの具体的提案） # サブエージェントへの指示（オーケストレーション方針） - 必要に応じて、Gensparkの「並列検索（Instagramの最新トレンド）」「AI Sheet Agent（ハッシュタグ・競合分析データ整理）」「AI Slides / Document Agent（投稿カルーセル・動画台本の構成作成）」を自律的に起動・連携させてください。 - 各サブエージェントの成果を1つの美しいマーケティング戦略レポートに統合して提示してください。 # 動作フロー ユーザーから情報が入力されたら、まず最初に「どのようなアカウント戦略を組み立て、どのサブエージェントを起動するか」の概要を提示し、その後自動で詳細な作成・分析に移ってください。

Chapter 07

達人になるための Tips

GenSparkをフル活用して「AIの社長」になるためのポイントです。

🎯

最初のゴールを具体的に書く

「SNS運用を改善したい」より「20代女性向けコスメブランドのInstagramで3ヶ月でフォロワーを1000人増やしたい」のように具体的に書くと、AIの精度が大幅に上がります。

🔍

実行プランを必ず確認する

エージェントが実行プランを提示したら、「このステップは不要」「このエージェントも追加して」と調整してから実行させましょう。この一手間でアウトプットの質が変わります。

📁

ファイル学習は最初にやっておく

カスタムエージェントを作ったら、まずRAG（ファイルアップロード）を設定しましょう。あなたの過去データを学習させることで、他のAIとは別次元の精度になります。

🔗

エージェントはURLで共有できる

作成したカスタムエージェントはURLで共有可能。チームメンバー全員が同じエージェントを使えるので、クオリティの統一ができます。

📊

Sparkpageを活用して成果物を共有

エージェントが出力するSparkpageは美しいWebページ形式で、そのままクライアントや上司への提出資料として使えます。

🎨

用途別にエージェントを複数作る

「Instagram用」「資料作成用」「リサーチ用」など、目的別に別々のカスタムエージェントを作っておくと、毎回ゼロから指示する手間が省けます。

タスクのためだけにリアルタイムで自動生成される「使い捨て」の専門エージェントです。事前に登録されているわけではなく、そのタスクに最適な役割を持つAIがその場で作られ、タスク完了後は消滅します。ユーザー側で意識する必要はなく、リードAIが自動的に判断して使います。
ファイルサイズや件数の上限はプランによって異なります。一般的には1ファイルあたり数MB〜数十MBが目安です。大きなファイルは内容を要約したものや、必要な部分だけを抜き出したファイルに分割してアップロードすることで精度が上がります。
はい、可能です。保存したカスタムエージェントにはURLが発行されます。そのURLをチームメンバーに共有するだけで、同じエージェントを全員が使えるようになります。これによりチーム全体のアウトプット品質を統一できます。
エージェントの成果物を美しいWebページ形式にまとめたものです。調査結果・戦略・提案が視覚的に整理されており、そのままURLで共有できます。クライアントへの提案書や社内報告として活用できる品質です。
現在、Call for Me は英語が主な対応言語です。日本のビジネスへの発信には対応していますが、会話は英語で行われます。日本語での自然な通話対応はまだ発展途上です。将来的な日本語対応が期待されています。予約や問い合わせ先が英語対応している場合は今すぐ活用できます。
テキストから動画・画像から動画を生成できます。Sora 2、Kling V2.5、Gemini Veo 3.1 など14以上のモデルが使え、AIが内容に応じて最適なモデルを自動選択（MOAスマート選択）します。クリップの延長・リップシンク・タイムライン編集も対応しており、SNS向けショート動画の作成に特に便利です。
現在公式に対応しているのは Gmail / Google Workspace と Outlook / Microsoft 365 の2系統です。GenSpark のワークスペース設定画面から連携を許可するだけで使い始められます。複数アカウントを同時接続して一元管理することも可能です。Yahoo!メールなど他サービスへの対応は現時点では未確認のため、公式サイトでご確認ください。
初期設定では「下書き生成のみ」モードになっており、AIが自動送信することはありません。まずはAIが生成した下書きを確認・編集してから送信する運用で始めるのが推奨です。慣れてきたらカスタムルールで特定条件の自動送信を有効にすることもできます。
